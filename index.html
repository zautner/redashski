Snapshot your Redash queries locally

A Chrome extension that captures Redash query outputs on demand. Keep a per-tab history of table data and chart visuals, compare results, and export to CSV — all without data leaving your browser.

Features

Everything stays local

Capture, compare and export Redash query data without any of it leaving your browser.

On-demand snapshots

Snap the current Redash query state with one click. Captures table data, metadata, and chart visuals.

Charts & tables

SVG and canvas chart captures plus full table data with columns and rows, all stored locally.

Side panel workspace

Browse history, switch between Table and Charts tabs, and compare snapshots in the Chrome side panel.

Per-tab FIFO queue

Each tab maintains its own snapshot history. Queue size is configurable from 1 to 100 entries.

CSV export

Export any table snapshot as CSV or copy it directly to your clipboard.

Zero data leakage

Data never leaves the browser. No analytics, no telemetry, no cookies. An 8 MB quota guard auto-evicts old snapshots.

Get started

Up and running in 4 steps

Redashski is loaded as an unpacked Chrome extension. No store installation required.

1

Clone the repo

Download or git clone this repository to your machine.

2

Load in Chrome

Open chrome://extensions/, enable Developer mode, click Load unpacked, and select the repo folder.

3

Configure URLs

Open Settings in the popup and add your Redash URL prefixes (e.g. https://redash.example.com/). The browser will prompt for host permission.

4

Start snapping

Navigate to a Redash query page, click Snap in the popup or side panel, and your results are captured locally.

Architecture

How it's built

Chrome Manifest V3 extension with no build step, no external dependencies.

content/

Injected into Redash pages. Detects and captures table + chart data from the DOM.

background/

Service worker that routes messages, manages the per-tab storage queue, and syncs icon state.

ui/popup/

Compact popup for quick actions: snap, toggle side panel, open settings.

ui/sidepanel/

Main workspace. Browse snapshot history, switch Table/Charts tabs, export CSV.

shared/

Shared constants: storage keys, message types, Redash DOM selectors, queue limits.
