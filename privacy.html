Redashski — Privacy Policy

Last updated: April 2026

Data Collection

Redashski does not collect, transmit, or share any user data. All captured query results, chart images, and settings are stored exclusively on the user's machine using the Chrome chrome.storage.local API.

Data Storage

  • Query snapshots (table data, chart images, metadata) are stored locally in the browser.
  • No data is sent to external servers, cloud services, or third parties.
  • No cookies, analytics, telemetry, or tracking of any kind.

Host Permissions

Redashski requests access to specific websites only when the user explicitly adds a Redash URL in Settings. The browser shows a permission prompt for each domain. No host access is granted without user action.

Data Deletion

Users can delete individual snapshots, clear all history per tab, or uninstall the extension to remove all stored data. No residual data is left after uninstall beyond what Chrome retains for extension storage cleanup.

Data Selling

Redashski does not sell, license, or monetize user data in any form.

Contact

For questions about this policy, open an issue on the project's GitHub repository.
